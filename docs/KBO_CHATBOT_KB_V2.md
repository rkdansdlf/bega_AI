KBO 리그 챗봇 지식 베이스 V2.0: 맥락적 대화 및 실시간 데이터 연동 프로토콜

1. 서론: 챗봇의 페르소나 및 진화 방향
본 문서는 KBO 리그 야구 플랫폼 챗봇을 위한 심화 지식 베이스다. 기존의 역사적 서사에 더해, 2025년 시즌의 정량적 지표와 시스템이 실시간으로 데이터를 가져올 수 있는 기술적 앵커(Anchor), 그리고 사용자에게 이미지를 설명하듯 대화할 수 있는 시각적 묘사 데이터를 추가했다. 이를 통해 챗봇은 "누가 이겼어?"라는 질문에 단순히 승패만 답하는 것이 아니라, "LG가 4-1로 승리하며 통합 우승을 확정했습니다. 특히 김현수가 5할대 타율로 시리즈를 지배했죠."라고 맥락 있는 답변을 제공할 수 있다.

2. 실시간 정보 연동을 위한 데이터 앵커 (Real-Time Anchors)
챗봇이 사용자의 질문 시점에 맞춰 가장 최신의 정보를 제공하기 위해, 외부 데이터베이스나 API와 연결되어야 할 핵심 포인트들이다. 개발팀은 아래 구조를 참고하여 시스템을 연동해야 한다.

2.1 실시간 경기 및 선수 데이터 (Live Data Points)
사용자가 "지금 점수 몇 대 몇이야?", "오늘 선발 누구야?"라고 물을 때 호출되어야 할 데이터 구조다.

경기 일정/결과 API (GET /game/schedule):
필수 필드: 경기 날짜, 시간, 홈/어웨이 팀, 현재 이닝, 점수, 구장 정보.
트리거 질문: "오늘 야구 해?", "지금 한화 경기 점수 좀."

선수 상세 기록 API (GET /player/info/{id}):
필수 필드: 시즌 타율/방어율, 최근 5경기 기록, 당일 라인업 포함 여부.
트리거 질문: "오늘 김현수 선발이야?", "안현민 요즘 잘해?"

순위 정보 API (GET /team/ranking):
필수 필드: 순위, 승/무/패, 승률, 게임차, 연속 승패 기록.
트리거 질문: "지금 5위 싸움 어떻게 돼가?", "롯데 몇 위야?"

2.2 챗봇 응답 로직 예시 (Pseudo-Code)

5. 팬 문화 및 응원가 (Contextual Culture)
데이터만으로는 설명할 수 없는 야구장의 열기를 전달한다.

돌아온 클래식, '엘도라도' (삼성): 저작권 문제로 사라졌던 삼성 라이온즈의 전설적인 응원가 '엘도라도'가 2024년부터 부활해 2025년 포스트시즌 대구 구장을 웅장하게 울렸습니다. 삼성 왕조 시대를 상징하는 곡입니다.
부산의 심장, '부산갈매기' (롯데): 2023년 저작권 합의 후 공식 부활하여, 사직야구장의 떼창 문화를 주도하고 있습니다.
2025년의 앤썸, '아파트(APT)': 로제와 브루노 마스의 히트곡이 KBO 응원가로 역수입되었습니다. "아~파트 아파트!"라는 중독성 있는 도입부가 경기장의 텐션을 올리는 데 쓰입니다.

7. 챗봇 답변 시나리오 예시 (Role-Playing)
Q: "한화 이글스 홈구장 어디야?"
나쁜 답변: "대전입니다."
좋은 답변 (맥락적): "2025년부터 한화 이글스는 새로운 홈구장인 **'대전 한화생명 이글스파크'**를 사용합니다. 기존 한밭야구장 옆에 지어졌으며, 비대칭 펜스와 복층 불펜이 특징인 최신식 구장입니다."

Q: "올해 신인왕 누구야?"
좋은 답변: "KT 위즈의 안현민 선수가 유력합니다. 신인임에도 타율 2위(0.334)와 출루율 1위(0.448)를 기록하며 '수원 고릴라'라는 별명답게 리그를 폭격했거든요."

Q: "박찬호 요즘 어디서 뛰어?"
맥락 파악: (MLB 투수 박찬호인지, 현역 타자 박찬호인지 구분 필요)
답변: "현역 내야수 박찬호 선수를 말씀하시는 거라면, KIA 타이거즈에서 활약하다가 2025 시즌 종료 후 FA 자격을 얻어 두산 베어스와 4년 80억 원에 계약하며 팀을 옮겼습니다."

결론: 이 문서는 KBO 챗봇이 단순 검색기를 넘어, 야구 팬과 '대화'가 통하는 친구가 되기 위한 필수 지침서다. 개발팀은 상기된 API 앵커를 우선적으로 확보하고, 시나리오 작가는 시각적 묘사와 응원가 등의 감성 데이터를 답변에 녹여내야 한다. 이제 챗봇을 업그레이드 해보자.
